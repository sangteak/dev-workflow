## 주요 발견 (본체 검증 완료)

1. **[확인] 결정 규범 문서(decision-flow.md)의 「서브에이전트 출력 재구성 책임」 절(149~153행)이 "고쳐 쓰기 금지"와 정면 충돌** (🛠️) — 개정·스윕 목록 필수 편입. 스윕 누락 시 구·신 규범 공존이 최대 실패 모드.
2. **[확인+경로 보정] agent-roles.md의 "verbatim 문장 수정 금지" 계약** (🎯) — Contrarian 교체의 실제 대상은 templates.md의 래퍼 프롬프트이므로 래퍼만 수정하면 계약 유지. 이번 세션 라운드들이 이 방식으로 실증됨.
3. **[✔] 태그의 보증 간극** (🎯) — 실제 보증(증거 대조 통과)과 지각 보증(완전 검증) 사이 간극이 최대 UX 리스크 → 세션 최초 출현 시 1회 뜻 고지로 완화.
4. **잘못 붙은 [✔]가 결정 근거일 때의 구멍** (🔍) — 프로젝트 내 출처 근거 의무 확대(수용)로 대부분 폐쇄. "결정 근거 [✔] 표본 재확인"은 관측 후 조정 후보로 보류·승계 (확정 결정 "표시 기반 선별 + 관측 조정"의 자연 경로).
5. **압축 생존 문제** (🛠️) — 등급 판정 근거·재실행 이력·관문 통과 이력은 컨텍스트 압축 후 소실 가능 → 결정 규범 문서의 상태 보존 절에 등재.
6. **규범 총량 증가의 역설** (3인 수렴) — 상시 주입 문서 비대는 준수율을 낮춤 → 신설 상세는 references/ 분리, 원본에는 포인터만 (v1.13.0 분할 전례).

## 수용 가이드라인 (설계 문서 8절에 전문 수록)

- 규범 배치: 신설 상세 references 분리 + 원본 포인터 (3인 수렴)
- 스윕: 대상 문구 목록 착수 전 확정 (재구성 책임 절 포함), 목록을 커밋에 남김, sweep + bump-version --check 릴리스 게이트
- 등급: 판정 질문 "토론·조사가 선행됐는가" 단일화 + 경계 사례 예시 2~3개, 낮은 등급 도주 방지
- 길이 역화 방지: 4칸 각 줄 1줄 상한 / "근거 3은 상한이지 채움 목표 아님" / 근거 부족 시 사유 1줄 대체 탈출구
- 가시화: 재실행 시 "발언 1건 형식 위반 — 다시 받는 중" 표시 / 발언 제외 고지 문구 실물 규정 / 워커 인용 미첨부 강등 고지 / 소급 반증 시 대상 발언 이름 지칭 형식
- [✔] 태그: 세션 최초 출현 시 뜻 1회 고지 (블라인드 최초 고지 패턴)
- 관문: 수행 주체 본체 고정 (워커 하향 금지) / 네트워크 차단 시 외부 축 미확인 강등 명시
- 관측 파일: 부재 시 조용히 스킵 / 항목 서식 1줄 / 파일 머리 용도 1줄 / 고지 첫 회만 / 도입 후 기록 건수 실측
- feedback 명령: 텍스트 초안 생성까지만 (자동 업로드·gh 실행 없음)
- 압축 생존 상태 등재: 등급 판정 근거·재실행 이력·관문 이력 → 상태 보존 절

## 검증 중 사용자 교정 (즉시 채택 — 개정 범위 편입)

1. **가이드라인 슬래시 압축 재위반 적발** — 본체가 판정 중계 시 줄바꿈 목록을 슬래시로 압축. 교훈: 길이는 생산 규격이 해결, 전달 압축 금지. 줄바꿈 규칙은 중계·요약 문면에도 적용 명시
2. **주석 마커 단일화** — `- ↳` 이중 마커 금지, `↳` 단독 + 들여쓰기
3. **절 기호(§) 등 법조문식 표기를 사용자 대면 문면에서 금지** — "라벨 병기" 절충 기각, 문서 이름+절 이름을 말로 풀어 쓴다. 언어 규범 조항 추가

## 전제 점검 (3인 공통)

설계 전체가 "문서 규범을 모델이 지킨다 + 위반은 본체가 알아챈다"에 기댐 — 자동 검사기 없음. 완화 장치: 재실행 규칙(생산 측), 관측 파일(감지 측), 릴리스 시점 기계 검사(sweep), 압축 생존 등재(상태 측). 이 연결이 성패를 가름.
