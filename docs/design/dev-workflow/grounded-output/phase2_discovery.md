## 미정의 영역과 결정 (6건 + 파생 1건)

### 영역 1: 4요소 박스의 적용 등급 → 출신 2등급 + 빈 슬롯 "없음" 명시
- 토론·조사 선행 결정 = 4칸 상시 (내용 없으면 "없음"이라 명시 — 생략 금지)
- 토론 없는 표준 결정 = "다루는 문제" 슬롯 필수 + 이력 있는 슬롯만 추가
- 로컬 템플릿 결정·승인 게이트 = 기존 형식 유지 (§1·§8 기존 경계 승인)
- 등급 판정 조건은 기계적 술어만 (라운드·조사 이력 존재 여부) — 주관 술어("중요한 결정이면") 금지
- 근거: 일률 강제는 §8과 지시 충돌로 규범 불안정화, 빈 슬롯 강제는 내용 지어내기 유발, "없음" 명시는 지어내기와 "생략인지 미수행인지 구분 불능"을 동시 차단

### 영역 2: 생산 규격 수치 → 통일 정형 + 재실행 1회
- 모든 라운드 발언: 결론 1줄 + 근거 불릿 3개 이내 (+ 전제 반박 1개 별도)
- §7 블라인드 정형과 동일 수치로 규범 통일. 이번 세션 라운드 4회 실측으로 충분성 확인
- 위반 시 재실행 1회 (본체 고쳐 쓰기 금지), 재위반 시 해당 발언 제외 + 1줄 고지

### 영역 3: 본체 주석(↳) → 예외 주석 방식
- 규약 명문화: "화면에 도달한 [✔] 태그 = 본체 관문(증거 대조) 통과"
- ↳ 주석은 3경우만: 반증([✘]) · 확인 불가 승격([⚠️]) · 본체 재확인으로 새 사실 추가
- 근거: 전수 주석은 발언 길이 2배 (길이 문제 회귀), 규약이 구분 불능 우려 해소

### 영역 4: 외부 문서 워커 → 인용 첨부 의무
- 워커 반환 규격: 주장별 판정(확인/반증/못 찾음) + 출처 주소 + 해당 대목 원문 인용
- 인용 없는 보고는 미확인([⚠️]) 취급 — 신뢰 지점은 워커 판정이 아니라 인용문 대조
- 근거: 경량 워커 허위 보고 실사례 (tasks/lessons.md 2026-07-14) — 페르소나·워커 동일 원칙

### 영역 5: 관측 + 형식 검사 훅 → 훅 미도입 + 소비자 피드백 루프 3부 구성
- 형식 검사 훅(교정형·관측 전용 모두) 미도입 — 교정형은 지연+굿하트 위험, 관측 전용은 "자동 루프성 기능은 플러그인 밖" 기존 결론과 충돌. 자동 검사가 필요해지면 저장소 릴리스 시점 검사(스윕·테스트)에 얹는다
- 소비자 피드백 루프 (사용자 제안·채택):
  1. **로컬 관측 파일**: 소비 프로젝트에 플러그인 전용 관측 파일 (가칭 `tasks/dw-observations.md`) — 플러그인 UX 사고 사례만 (되묻기·정정·형식 불편·개선 아이디어), 고정 서식 (날짜/상황/불편/기대 동작). lessons.md와 분리 (추출 시 프로젝트 기밀 격리 + 고정 서식)
  2. **로컬 적용**: 세션 시작 교훈 검토 단계가 이 파일도 읽음 — 다음 세션부터 로컬 반영
  3. **추출·제보 명령**: `/dev-workflow:feedback` — 관측 파일을 Git 이슈용 제보 초안으로 정리 출력. 자동 업로드 없음 (기밀 검토 후 사용자가 직접 등록), 관리자가 이슈를 규범에 반영
- 기록 트리거: 자동 아님 — 사용자가 지적했거나 기록을 요청했을 때 1줄씩 (고지 후)
- 관측 대상: 형식 준수율이 아니라 사용자 사고 사례 (박스 내용 되묻기 / 박스와 반대 정정 / 재확인 누락)
- 전제 확인: 소비자 관측의 자동 회수 통로는 없음 (로컬 마켓플레이스 배포·단일 관리자) — 제보는 수동 경로만 성립

### 영역 6: 문서 개정 범위 + 버전 → 5개 영역 + MINOR
1. decision-flow.md — 스켈레톤 4요소·출신 등급·"없음" 명시 개정, 검증 규범 신설 (3출처·전치 태그·관문·표시 기반 재확인·워커 인용 의무·"[✔]=관문 통과" 규약), §7 블라인드 프롬프트 출처 의무 추가
2. brainstorming — Contrarian 프롬프트 교체 ("가정 지적+반사실" → "출처 표시+분량 상한" 정형, 벽 텍스트 발원지), 대화형 라운드 중계를 원문+주석 방식으로
3. plan-stage·orchestrator — 프롬프트 조립·폴백 문구 동기화 + 구 스켈레톤 재서술 잔존 스윕 (규범 변경 스윕 관행, skills 밖 포함)
4. 신규 — 관측 파일 규격 + 세션 시작 교훈 검토 확장 + commands/feedback
5. 버전: MINOR (기존 사용자 프로젝트 docs/design 구조 무변 — 출력 형식·기능 추가만)

### 파생 논의: 형식 검사 훅 (사용자 질문에서 파생)
- 훅 = 실행 환경이 정해진 시점에 강제 실행하는 외부 스크립트 (모델 협조와 무관)
- 결정 박스 구조 검사·태그 검사·관측 수집이 기술적으로 가능하나, 내용의 질은 검사 불가
- 영역 5에 병합되어 미도입 확정

## 페르소나 피드백 결과 (주제별)

### 영역 1 [블라인드 라운드]
- 합의: 출신 기반 기계 판정 등급. 미합의 1건 (빈 슬롯 생략 🛠️ vs 상시 🎯) → "없음" 명시 절충으로 사용자 확정
- 🛠️ 전제 반박: 형식 검사기 부재 → 등급 설계가 지시문 한 문단으로 요약 안 되면 실패 (규격 단순성 제약)
- 🎯 전제 반박: 배너 무시 현상 (4칸 도배 → 건너뛰어 읽기) → 관측 항목으로 승계

### 영역 5 [대화형 라운드 · 비판: 🎯 Contrarian]
- 관측 대상 반전: "형식 준수율"(지키기 쉬운 것)이 아니라 "사용자 사고"(되묻기·정정)를 잰다 — 편향 표본은 아픈 순서로 정렬된 표본이라 조정 목적에 오히려 유효
- 런타임 훅 3택 기각: 교정형은 자기모순(기다림 증가)+굿하트, 관측 로그는 기존 결론 충돌. 제4의 자리 = 저장소 릴리스 검사
- 🛠️ 보강: 스윕·훅 테스트 인프라 기존재 — 신규 인프라 없이 가능
