## 미정의 영역 → 결정 사항

### 1. 구현 방식

**옵션 3가지:**
1. 스크립트 파일 추가 (hooks/ 또는 별도 디렉토리) — 확실하지만 플러그인 성격 변경
2. 스킬 내 인라인 Bash 지시 — 코드 없는 구조 유지, 탐색 스텝에 구체적 명령 명시
3. 스킬 지시문만 보강 — "실제 경로를 먼저 확인하라" 수준의 최소 변경

**결정:** 옵션 2 (스킬 내 인라인 Bash 지시)

**근거:**
- "코드 없는 플러그인" 정체성 유지
- lessons.md 교훈: "스킬 지시문과 실제 출력의 괴리" — 옵션 3은 Claude 재량 의존으로 위험
- 스크립트 추가(옵션 1)는 크로스 플랫폼 이슈, 디버깅 복잡도 증가

### 2. 신뢰성 기준

- 사용자 발언: "매번 디자인 디렉토리에 가서 뒤져봐야 하는 상황이 되면 플러그인 신뢰를 잃음"
- **기준:** 찾지 못하는 경우가 0이어야 한다 — 편의성이 아닌 신뢰성 문제

### 3. 적용 범위

- context-handling 단독이 아닌, `docs/design/` 경로를 참조하는 **모든 스킬**에 일관 적용
