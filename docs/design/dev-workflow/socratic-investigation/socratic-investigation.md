# 소크라틱 인터뷰 미답변 질문 조사 (socratic-investigation)

> SSOT | 카테고리: dev-workflow | 생성일: 2026-03-27

---

## 1. 개요

소크라틱 인터뷰(Step A, Step A-0) 질문 목록 제시 후, 사용자가 답변하지 않은 질문에 대해 서브에이전트가 코드베이스/설계 문서를 조사하여 잠정 답변을 제공한다. 사용자의 직접 답변 부담을 줄이면서도 사용자 통제권을 유지하는 것이 핵심이다.

---

## 2. 목표

- **GOAL-001**: 사용자가 답변하지 않은 소크라틱 질문에 대해 코드/문서 기반 잠정 답변을 제공하여 답변 부담을 줄인다
- **GOAL-002**: 사용자가 답변 여부를 통해 조사 범위를 자연스럽게 통제한다
- **GOAL-003**: 기존 brainstorming 워크플로우(Phase 0~3)와 완전 호환을 유지한다

---

## 3. 비목표 (Non-goals)

- Ouroboros 에이전트 역할 정의 변경
- 서브에이전트 프롬프트 구조 변경
- 조사 대상 질문의 자동 선별/분류 (사용자 통제 원칙 위반)
- 서브에이전트에 코드베이스 컨텍스트 사전 주입 (앵커링 편향)
- 조사 기능의 강제 실행 — 모든 질문에 직접 답변하는 경로 항상 보장

---

## 4. 설계

### 4.1 변경 범위

- `skills/brainstorming/SKILL.md` — 통합/정제 파이프라인 섹션 + Step A-0 후속 처리
- 다른 스킬 파일, Ouroboros 에이전트 정의는 변경하지 않음

### 4.2 핵심 플로우

```
[기존 — 변경 없음]
서브에이전트 → 질문 생성 → 통합 정제 (중복 병합, 그룹핑, 힌트)
                                    ↓
[변경] 질문 제시 + 안내 문구
                                    ↓
📋 핵심 질문 (N개):
1. [주제] 질문 내용
   > 힌트: ...
...
답변할 수 있는 질문에 답해 주세요.
답변하지 않은 질문은 코드/문서 조사를 진행합니다.
                                    ↓
[사용자 답변] "2번은 OO. 4번은 XX."
                                    ↓
[변경] 2-pass 처리
                                    ↓
Pass 1: 직접 답변 반영
  ✅ 답변 반영: 2번, 4번

Pass 2: 미답변 확인 + 조사
  미답변 질문이 있습니다: 1번, 3번, 5번
  코드/문서 조사를 진행할까요?
  1. 전부 조사
  2. 선택 조사 (번호 지정)
  3. 스킵 — 다음 라운드로
                                    ↓
[사용자 확인] "1" (전부 조사)
                                    ↓
[변경] 조사 실행
  Enhanced: 조사 서브에이전트 (단일, 일괄 처리)
  Standalone: 메인 컨텍스트 직접 조사
                                    ↓
── 🔍 조사 결과 ──────────────────
1. [질문]
   → [조사 내용 요약]
3. [질문]
   → [조사 내용 요약]
5. [질문]
   → [조사 내용 요약]
────────────────────────────────────
수정이 필요하면 알려주세요. 없으면 다음으로 진행합니다.
                                    ↓
[사용자 확인/수정/이의] → 확정 or 재조사
```

### 4.3 조사 서브에이전트 프롬프트 (Enhanced Mode)

```
아래 질문들에 대해 프로젝트의 코드베이스와 설계 문서를 조사하라.

탐색 전략:
1순위: docs/design 디렉토리 — 관련 설계 문서 검색 및 읽기
2순위: 코드베이스 — Grep/Glob/Read로 관련 코드 탐색

각 질문에 대해:
- 발견한 내용 요약
- 확신도 표시 (높음/중간/낮음)
- 근거 파일 경로 명시

조사 대상:
1. [원래 질문 1 내용]
3. [원래 질문 3 내용]
5. [원래 질문 5 내용]
```

### 4.4 Step A-0 (Ontologist) 지원

Step A-0의 본질 질문(Essence, Root Cause, Prerequisites, Hidden Assumptions)에도 동일한 미답변 조사 패턴을 적용한다. 단, A-0 출력 직후 1회만 실행.

### 4.5 모드별 분기

| 모드 | 조사 실행 주체 | 비고 |
|---|---|---|
| Enhanced (Path A/B) | Agent 도구로 별도 서브에이전트 | 메인 컨텍스트 보존 |
| Standalone (Path C) | 메인 컨텍스트에서 직접 Glob/Grep/Read | 서브에이전트 불가 환경 |

로직은 동일하며 실행 주체만 다르다. SKILL.md에서 1줄 분기.

## 5. 제약 조건

- **REQ-001**: Ouroboros 서브에이전트 정의(역할·프롬프트 구조) 변경 불가
- **REQ-002**: 자율 질문 필터링/자기해소 금지 — 미답변 질문만 조사 대상, 사용자가 답변 여부로 통제
- **REQ-003**: 미답변 질문에 대해 즉시 조사하지 않고 확인 단계를 거침 — 미답변 의도(조사/유보/무관) 구분
- **REQ-004**: 서브에이전트 사전 컨텍스트 주입 금지 — 앵커링 편향 방지
- **REQ-005**: 조사 서브에이전트는 메인 컨텍스트가 아닌 별도 서브에이전트로 실행 (Enhanced)
- **REQ-006**: 조사 결과는 확정이 아닌 잠정 답변으로 제시 — 사용자 최종 확인 필수
- **REQ-007**: Step A-0(Ontologist) 질문에도 동일한 미답변 조사 패턴 적용 — A-0 출력 직후 1회만
- **REQ-008**: Standalone Mode에서는 메인 컨텍스트에서 직접 조사
- **REQ-009**: 조사 결과 제시는 2-pass 방식 — 직접 답변 반영 먼저, 조사 결과 별도 블록
- **REQ-010**: 수정 대상은 brainstorming SKILL.md의 통합/정제 파이프라인 + Step A-0 후속 처리만

## 6. 수락 기준 (Acceptance Criteria)

- **AC-001**: 사용자가 답변하지 않은 질문에 대해 확인 후 조사를 진행한다 (미답변 → 확인 → 조사)
- **AC-002**: 조사 서브에이전트가 docs/design 우선, 코드베이스 차선으로 탐색한다
- **AC-003**: 조사 결과는 자유 응답 형태로 확인 — 수정 필요 시 사용자 코멘트, 없으면 확정
- **AC-004**: 이의 제기 시 재조사 또는 사용자 직접 답변으로 교체 가능
- **AC-005**: 직접 답변 먼저 반영 → 미답변 조사 결과 별도 블록 (2-pass)
- **AC-006**: 질문 제시 시 안내 문구 포함: "답변할 수 있는 질문에 답해 주세요. 답변하지 않은 질문은 코드/문서 조사를 진행합니다"
- **AC-007**: 모든 질문에 직접 답변하면 조사 단계 자동 스킵
- **AC-008**: 모든 질문에 직접 답변해도 워크플로우 정상 동작
- **AC-009**: 기존 brainstorming Phase 0~3 흐름 정상 동작
- **AC-010**: Step A-0(Ontologist) 질문에도 미답변 조사가 동작
- **AC-011**: Standalone Mode(Ouroboros 미설치)에서도 메인 컨텍스트 직접 조사로 동작

## 7. UX 설계

### 7.1 안내 문구

질문 목록 하단에 한 줄 안내:
```
답변할 수 있는 질문에 답해 주세요.
답변하지 않은 질문은 코드/문서 조사를 진행합니다.
```

### 7.2 미답변 확인 단계

미답변 의도를 구분하기 위한 확인:
```
미답변 질문이 있습니다: 1번, 3번, 5번
코드/문서 조사를 진행할까요?

1. 전부 조사
2. 선택 조사 (번호 지정)
3. 스킵 — 다음 라운드로
```

### 7.3 조사 결과 제시

직접 답변과 명확히 분리된 별도 블록:
```
── 🔍 조사 결과 ──────────────────
1. [질문]
   → [조사 내용 요약]
────────────────────────────────────
수정이 필요하면 알려주세요. 없으면 다음으로 진행합니다.
```

### 7.4 이의 제기 대응

사용자가 조사 결과에 이의를 제기할 경우:
- 재조사 (범위 확장/변경)
- 추가 조사
- 사용자 직접 답변으로 교체

## 8. 의사결정 기록

| 결정 | 채택 | 기각된 대안 | 근거 |
|---|---|---|---|
| 조사 트리거 방식 | 미답변 + 확인 | 자율 해소, 명시 형식(`조사: 1,3`) | 오탐 방지 + 학습 비용 0 + 미답변 의도 구분 |
| 서브에이전트 컨텍스트 | 사전 주입 없음 | Pre-context 주입 | 앵커링 편향 방지 |
| 조사 실행 방식 | 단일 서브에이전트 일괄 | 질문별 개별 서브에이전트 | SKILL.md 단순성 |
| 결과 확인 방식 | 자유 응답 | 3분기(맞음/보충/틀림) | UX 단순화 |
| 결과 제시 방식 | 2-pass 분리 | 혼합 제시 | 맥락 보존, 혼란 방지 |
| 탐색 전략 | docs/design 1순위 → 코드 2순위 | 코드만 / 문서만 | 설계 문서 신뢰도 우선 |
