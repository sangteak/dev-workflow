## 요구사항

### 배경

소크라틱 인터뷰 서브에이전트(Ouroboros 기반)가 생성하는 질문 중 일부는 코드베이스/설계 문서 조사로 답변 가능하다. 현재는 이러한 질문도 모두 사용자가 직접 답변해야 하며, 이는 불필요한 부담을 만든다.

### 핵심 결정

| 의사결정 | 결과 | 근거 |
|---|---|---|
| 자율 필터링/자기해소 | ❌ 제외 | 오탐(false-confidence) 위험이 Critical |
| Pre-context 주입 | ❌ 제외 | 앵커링 편향 + 문서 부재 시나리오 |
| 사용자 명시 지정 (`조사: 1, 3`) | ❌ 제외 (UX 개선) | 새 형식 학습 비용, 불자연스러움 |
| **미답변 = 조사 트리거** | ✅ 채택 | 기존 답변 방식과 동일, 학습 비용 0 |

### 확정 플로우

```
[기존] 서브에이전트 → 질문 생성 → 통합 정제

📋 핵심 질문 (N개):
...
답변할 수 있는 질문에 답해 주세요.
답변하지 않은 질문은 코드/문서를 조사하여 잠정 답변을 드립니다.

[사용자] "2번은 OO. 4번은 XX."

✅ 답변 반영: 2번, 4번
⏳ 미답변 질문(1번, 3번, 5번) 조사 진행...

── 🔍 조사 결과 ──────────────────
1. [질문] → [조사 내용 요약]
3. [질문] → [조사 내용 요약]
5. [질문] → [조사 내용 요약]
────────────────────────────────────
수정이 필요하면 알려주세요. 없으면 다음으로 진행합니다.
```

### 제약 조건

1. Ouroboros 서브에이전트 정의(역할·프롬프트 구조) 변경 불가
2. 자율 질문 필터링/자기해소 금지 — 미답변 질문만 조사 대상
3. 서브에이전트 사전 컨텍스트 주입 금지 — 앵커링 편향 방지
4. 조사 서브에이전트는 메인 컨텍스트가 아닌 별도 서브에이전트로 실행
5. 조사 결과는 확정이 아닌 잠정 답변 — 사용자 최종 확인 필수
6. 수정 대상: brainstorming SKILL.md의 통합/정제 파이프라인 섹션만
7. Step A-0(Ontologist) 흐름 변경 불가
8. 조사 결과 제시는 2-pass 방식 — 직접 답변 먼저, 조사 결과 별도 블록

### 제외 항목

| 항목 | 이유 |
|---|---|
| Ouroboros 에이전트 역할 변경 | 범위 외 — 질문 생성은 그대로 유지 |
| 서브에이전트 프롬프트 구조 변경 | 앵커링 편향 위험 |
| 조사 대상 자동 선별/분류 | 오탐 위험 Critical |
| 코드베이스 컨텍스트 사전 주입 | 앵커링 편향 + 적용 가능 시나리오 제한적 |
| 조사 강제 실행 | 모든 질문 직접 답변 경로 보장 필수 |

### AC (수락 기준)

1. 사용자가 답변하지 않은 질문이 자동으로 조사 대상이 된다 (미답변 = 조사 트리거)
2. 조사 서브에이전트가 docs/design 우선, 코드베이스 차선으로 탐색한다
3. 조사 결과는 자유 응답 형태로 확인 — 수정 필요 시 코멘트, 없으면 확정
4. 이의 제기 시 재조사 또는 사용자 직접 답변으로 교체 가능
5. 직접 답변 먼저 반영 → 미답변 조사 결과 별도 블록(2-pass)
6. 안내 문구: "답변하지 않은 질문은 코드/문서 조사를 진행합니다" 포함
7. 모든 질문에 직접 답변하면 조사 단계 자동 스킵
8. 기존 brainstorming Phase 0~3 흐름 정상 동작
