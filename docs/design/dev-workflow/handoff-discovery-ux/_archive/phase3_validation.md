# Phase 3: 검증 (Validation) — handoff-discovery-ux

## TD 기술 검토 결과

### A. 단일 패스 탐색 + 출력 억제
- **판단:** 실현 가능하나 Claude 행동 제어에 한계 있음
- **가이드라인:** 절차를 "1. 탐색(출력하지 않는다) 2. 아래 템플릿만 출력한다"로 구조화하여 출력 시점을 하나로 제한
- **리스크:** 중간 — 완전 제거 불가, 발생 빈도 감소 가능
- **대응:** 금지 출력 패턴 명시 + 절차 단순화 + 출력 시점 단일화

### B. 파일 조합 기반 라벨 추론
- **판단:** 실현 가능
- **가이드라인:** frontmatter 파싱은 파일 상위 10줄로 제한, status 필드만 추출
- **리스크:** 낮음

### C. orchestrator Step 2 축소
- **판단:** 실현 가능, 구조적으로 더 깔끔
- **가이드라인:** 탐색 로직 완전 제거, context-handling invoke만 남김. "진행 중인 작업 없음" 결과도 context-handling이 처리
- **리스크:** 낮음

### D. 복구 흐름 메시지 분기
- **판단:** 실현 가능
- **가이드라인:** 메시지 템플릿을 스킬에 삽입, "아래 템플릿을 그대로 사용한다" 명시
- **리스크:** 낮음

### E. docs/design/ 미존재 처리
- **판단:** 실현 가능
- **가이드라인:** context-handling은 "작업 없음 출력"까지만 담당, orchestrator가 후속 질문 처리
- **리스크:** 낮음
