# Phase 2: 발견 (Discovery) — persona-feedback-loop

## 확인된 요구사항

### 1. 피드백 루프 출력 형식
- 의견 본문: 자유 형식 (페르소나 이모지 + 이름 접두사만)
- 라운드 종료 시: `**합의:**` 한 줄 마커 + 결론 요약
- 미합의 시: `**미합의:**` + 쟁점 요약
- 정형 태그(찬성/반대 등) 사용하지 않음

### 2. 토론 리드 선정 방식
- 주제별 유동 선정 (고정 아님)
- Claude가 주제-페르소나 전문성 매칭으로 자동 선정
- 리드 선정 시 이유 한 줄 명시
- 연속 2회 리드 제한: 같은 페르소나가 연속으로 리드하지 않음

### 3. 영향 범위
- 주 수정 대상: `brainstorming/SKILL.md` — 피드백 루프 메커니즘 추가
- 수정 불필요: `workflow-orchestrator/SKILL.md`, `development-principles/SKILL.md`
- 별도 이슈로 분리: `persona-resolution/SKILL.md`의 기본 페르소나 범용화

### 4. plan-stage Feedback Loop와의 관계
- 완전 별도 유지, 의존성 없음
- 각 스킬이 자기 피드백 루프 규칙을 독자적으로 소유
- 현재 값이 같더라도 의존성을 만들지 않음
- 각 스킬이 자기 단계에 맞게 독립적으로 진화 가능

### 5. phase 파일 기록 방식
- phase 파일에 합의/미합의 결론만 기록 (토론 과정 제외)
- 기존 phase 파일 "포함 내용"에 한 섹션 추가
- 주제별 그룹핑 형식:
  ```
  페르소나 피드백 결과:
  - [주제 1]: 합의 — (결론 요약)
  - [주제 2]: 미합의 — (쟁점 요약)
  ```

## 페르소나 피드백 결과
- [출력 형식]: 합의 — 자유 형식 + 합의/미합의 마커
- [토론 리드]: 합의 — 주제별 유동, 자동 선정, 연속 2회 제한
- [영향 범위]: 합의 — brainstorming/SKILL.md만 수정
- [plan-stage 관계]: 합의 — 완전 별도 유지, 의존성 없음
- [phase 파일 기록]: 합의 — 합의/미합의 결론만, 주제별 그룹핑
