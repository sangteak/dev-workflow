# Plan: persona-feedback-loop

## 개요
`brainstorming/SKILL.md`에 페르소나 피드백 루프 메커니즘을 추가한다.
수정 대상은 이 파일 하나뿐이다.

---

## Task 1: 피드백 루프 공통 규칙 섹션 추가

**위치:** `brainstorming/SKILL.md` — 국면 0과 국면 1 사이에 새 섹션 삽입
**내용:** 모든 국면에 공통 적용되는 피드백 루프 규칙을 정의

추가할 규칙:
- 매 주제/질문마다 피드백 루프 발동
- 최대 3회 루프 제한, 초과 시 사용자 승인
- 라운드 카운터 `[라운드 N/3]` 출력
- 비판적 역할은 라운드마다 다른 페르소나가 맡는다 (라운드 로빈)
- 라운드 시작 시 비판적 역할 페르소나를 명시
- 비판 시 반드시 대안 또는 출구를 함께 제시
- 비판적 사고 방향성: 비판 60% / 건설적 40%
- 자유로운 분위기 유지, brainstorming 발산적 성격 훼손하지 않음

**완료 기준:** 섹션이 추가되고 기존 스킬 구조가 유지됨

---

## Task 2: 토론 리드 규칙 추가

**위치:** Task 1에서 추가한 피드백 루프 섹션 내부
**내용:** 토론 리드 선정 및 역할 규칙

추가할 규칙:
- 매 주제 시작 시 사용자에게 토론 리드를 묻는다 (선택지에 "없음" 포함)
- 리드 선택 시: 선정 이유 한 줄 명시, 비판적 비중 약간 높게
- "없음" 선택 시: 모든 페르소나 동등 참여 (기본 비율)
- 같은 페르소나가 연속 2회 리드하지 않음

**완료 기준:** 토론 리드 규칙이 피드백 루프 섹션에 포함됨

---

## Task 3: 출력 형식 정의

**위치:** Task 1에서 추가한 피드백 루프 섹션 내부
**내용:** 피드백 루프의 출력 형식 템플릿

정의할 형식:
- 의견 본문: 자유 형식 (페르소나 이모지 + 이름 접두사)
- 라운드 종료 시: `**합의:**` 또는 `**미합의:**` 마커 + 결론 요약
- 정형 태그(찬성/반대 등) 사용하지 않음
- 라운드 카운터 포함

**완료 기준:** 출력 형식 예시가 포함됨

---

## Task 4: 국면별 방향성 가이드 추가

**위치:** Task 1에서 추가한 피드백 루프 섹션 내부
**내용:** 국면별 비판/건설적 방향성 조절 가이드

추가할 가이드:
- 국면 1 (탐색): 비판보다 호기심으로
- 국면 2 (발견): 기본 비율 유지
- 국면 3 (검증): 리스크를 놓치지 말 것
- 강제 비율 없음, 방향성 가이드로 자연 조절

**완료 기준:** 국면별 가이드가 포함됨

---

## Task 5: 기존 국면 2 규칙 수정

**위치:** `brainstorming/SKILL.md` 국면 2 섹션 (82~87행 부근)
**내용:** 기존 "충돌/보완" 규칙을 피드백 루프 참조로 대체

변경 전:
- "도메인 페르소나들이 서로 충돌/보완하면서 요구사항을 구체화한다"
- "충돌 시 양측 입장을 명확히 제시하고 사용자가 결정한다"

변경 후:
- "도메인 페르소나들이 피드백 루프를 통해 요구사항을 구체화한다 (피드백 루프 규칙 참조)"
- 기존 충돌 처리 규칙은 피드백 루프 공통 규칙으로 흡수

**완료 기준:** 국면 2 규칙이 피드백 루프와 정합성 있게 수정됨

---

## Task 6: phase 파일 포함 내용에 피드백 결과 섹션 추가

**위치:** `brainstorming/SKILL.md` 각 국면의 "포함 내용" 목록
**내용:** 각 phase 파일에 피드백 루프 결과 기록 항목 추가

추가할 항목 (각 국면 공통):
```
- 페르소나 피드백 결과 (주제별 합의/미합의 요약)
```

형식:
```
페르소나 피드백 결과:
- [주제 1]: 합의 — (결론 요약)
- [주제 2]: 미합의 — (쟁점 요약)
```

**완료 기준:** phase1, phase2, phase3 각각의 포함 내용에 항목 추가됨
