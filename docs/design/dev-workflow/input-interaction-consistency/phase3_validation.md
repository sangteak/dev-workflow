## 재협의 발생 항목

### 자연어 폴백 허용 여부
- **발견 경위:** 🔍 Ecosystem Analyst가 "번호 강제 시 습관적 자연어 응답 처리" 리스크 지적
- **선택지:** 엄격 모드(번호만) vs 관용 모드(번호 우선 + 자연어 수용)
- **최종 결정:** 관용 모드 채택

### 규칙 대상 재정의
- **발견 경위:** 관용 모드 논의 중 규칙의 본질 재정의
- **최종 결정:** 규칙의 대상은 **Claude의 출력 형식**이지, 사용자의 입력 방식이 아님. 사용자는 번호 또는 자연어로 자유롭게 응답 가능. 의미 모호 시에만 재확인.

## TD 기술 가이드라인

1. AskUserQuestion 금지를 orchestrator 최상위(CRITICAL 섹션)에 배치하여 우선순위 확보
2. 각 스킬 파일의 기존 프롬프트 예시를 새 형식으로 완전 교체하여 구 형식 참조 제거
3. "AskUserQuestion 도구를 사용하지 않는다" — 서술형으로 기재
4. "사용자가 번호 또는 자연어로 응답할 수 있다. 의미가 모호한 경우에만 재확인한다" 명시
