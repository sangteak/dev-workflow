# Phase 1: Exploration — 입력 상호작용 일관성

> 생성일: 2026-03-17 | 불변 스냅샷

## 배경

dev-workflow 플러그인에서 사용자 입력을 받는 36개 지점이 4가지 서로 다른 패턴으로 혼재되어 있으며, 세션/환경마다 입력 방식이 달라져 사용자 혼란을 야기하고 있다.

## 탐색된 요구사항

### 핵심 요구사항
- **REQ-EXPLORE-001**: 모든 스킬에서 사용자 입력 방식을 통일한다
- **REQ-EXPLORE-002**: 스킬 문서에 입력 방식을 명시적으로 지정하여 Claude 재량 여지를 제거한다
- **REQ-EXPLORE-003**: 텍스트 기반 번호 목록을 표준 입력 방식으로 채택한다
- **REQ-EXPLORE-004**: AskUserQuestion 도구를 사용하지 않는다 (선택지 2~4개 제한으로 일관성 확보 불가)
- **REQ-EXPLORE-005**: 팀 배포 시에도 동일한 입력 경험을 보장한다

### 문제 현황
| 패턴 | 빈도 | 설명 |
|---|---|---|
| Yes/No 텍스트 확인 | 21건 | 국면 전환, 문서 작업 확인 등 |
| 번호/문자 선택 목록 | 8건 | 카테고리, HANDOFF, 재협의 등 |
| 자유 텍스트 입력 | 6건 | 페르소나 수정, OPEN_QUESTIONS 등 |
| AskUserQuestion 도구 | (미명시) | Claude 재량으로 사용 |

### 근본 원인
- 스킬 문서에는 "무엇을 물어볼지"만 정의, "어떤 도구/형식으로 물어볼지"는 미정의
- Claude가 비결정적으로 AskUserQuestion 도구 사용 여부를 결정
- 동일 사용자가 데스크톱(텍스트 기반)과 노트북(AskUserQuestion)에서 다른 경험

### 사용자 명시적 제외
- AskUserQuestion 도구 사용: 선택지 2~4개 제한으로 모든 상황에 적용 불가, 텍스트 기반과 혼합 시 여전히 불일관

## 페르소나 피드백 결과

### 주제: 입력 방식 선택 (3라운드)

**라운드 1 — 합의**: 입력 방식 통일 필요. AskUserQuestion 제약(2~4개)으로 전면 도구 사용 불가, 입력 유형별 명확한 기준 필요

**라운드 2 — 합의**: AskUserQuestion-first 하이브리드 전략이 현실적이나, 팀원 혼란의 원인이 "불일관성"이라면 텍스트 기반 통일도 유효

**라운드 3 — 합의**: 사용자가 두 방식을 직접 비교 체험 후, 텍스트 기반 번호 목록 통일을 최종 선택. AskUserQuestion 제한 없이 일관성 확보 가능
