## 탐색된 요구사항

### 핵심 요구사항

1. **설계 문서 크로스레퍼런스**: 다른 기능을 개발할 때, 이미 설계된 관련 기능의 문서를 자동으로 찾아 읽어와 컨텍스트로 활용
2. **트리거 방식**: 브레인스토밍/플랜 단계에서 사용자의 자연어 키워드로 트리거 (예: "aggro 참고해", "combat-base 기반으로")
3. **대상 문서**: 구현 완료된 설계 문서(`[기능명].md`)만 포함. `status: completed` 프론트매터 값으로 판단
4. **status 통일**: 작업 완료 시 `status` 값이 일관되지 않을 경우, `completed`로 통일하는 규칙 수립 필요

### 두 가지 로드 모드

5. **색인 모드 (메타데이터 목록)**
   - 트리거: "어떤 기능 있어?", "목록 보여줘", "관련된 거 뭐 있어?"
   - 동작: 모든 `[기능명].md` 프론트매터 스캔 → `status: completed` 필터 → 메타데이터 목록 제시
   - 용도: 브레인스토밍 초반 탐색, 의존성 관계 파악
   - 컨텍스트 비용 최소

6. **전체 로드 (깊은 참조)**
   - 트리거: "~~ 참고해서", "~~ 기반으로", "~~ 고려해서"
   - 동작: 키워드로 문서 매칭 → 전체 문서 컨텍스트 주입 → 페르소나가 해당 시스템을 이해한 상태로 참여
   - 용도: 기존 시스템과 깊이 연관된 새 기능 설계

### 자동 모드 판단

7. **발화 패턴 기반 자동 판단**
   - "~~ 참고해서/기반으로/고려해서" → 전체 로드
   - "어떤 기능/목록/관련된 거" → 색인 모드
   - 모호한 경우 → 사용자에게 확인
