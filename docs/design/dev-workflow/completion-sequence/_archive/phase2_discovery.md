# Phase 2: 발견 (Discovery)

## 미정의 영역 및 결정 사항

### 1. 마무리 선언의 형태
- **결정:** 자연어 감지 방식 ("마무리해줘", "wrap up" 등)
- 필요 시 명시적 명령어(`/finish` 등) 추가 가능성 열어둠

### 2. README 영향 여부 판단 기준
- **결정:** Claude가 변경 내용을 분석해서 자동 판단 후 사용자에게 확인

### 3. 마무리 시퀀스 중 실패 시 처리
- **결정:** 실패 지점에서 멈추고 사용자에게 알림 (커밋 차단)

### 4. 마무리 시퀀스 제어 위치
- **결정:** `workflow-orchestrator`에 종료 프로토콜 추가 (중앙 제어)
- 페르소나 토론 합의 내용:
  - 오케스트레이터 역할을 "세션 시작"에서 "워크플로우 라이프사이클 관리"로 확장
  - 실행은 기존 스킬(`document-consolidation` 등)에 위임, 오케스트레이터는 흐름 제어만
  - Superpowers 커밋 제안 억제는 컨텍스트 주입 방식 (DEVELOP 진입 시 전달)

## 기대하는 마무리 흐름

```
Superpowers 태스크 완료 → 사용자 마무리 선언 → 문서 취합 자동 실행 → README 영향 판단 → 커밋+푸시 제안
```

### 핵심 변경점
1. Superpowers의 커밋 제안 억제 — DEVELOP 진입 시 "태스크 완료 후 커밋 제안하지 말고 완료 보고만 하라" 컨텍스트 주입
2. 마무리 시퀀스 자동 진행 — 사용자가 마무리 선언하면 오케스트레이터가 문서 취합 → README 확인 → 커밋 제안을 순서대로 자동 실행
