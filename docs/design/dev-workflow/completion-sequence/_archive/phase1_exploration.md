# Phase 1: 탐색 (Exploration)

## 확정 페르소나
- 🔧 TD / 🏛️ Architect / 📋 PM

## 문제 정의
- 구현+테스트 완료 후, 문서 취합(document-consolidation)이 실행되기 전에 커밋 제안이 먼저 나옴
- 사용자가 매번 "아니, 마무리하고 커밋하자"라고 말해야 하는 마찰 발생
- 문서 취합 단계가 스킵되는 문제

## 탐색된 요구사항

### 마무리 시퀀스 정의
- 기대하는 순서: 구현+테스트 완료 → 사용자 마무리 선언 → 문서 취합 → README 업데이트 → 커밋+푸시
- 문서 취합은 커밋 전 필수 단계로 강제되어야 함

### 마무리 트리거
- 사용자가 명시적으로 마무리를 선언하는 방식 선호
- 테스트는 프로젝트마다 사용자 확인이 필요하므로 자동 감지 부적합

### 마무리 시 처리 항목
1. 문서 취합 (document-consolidation) — 항상 실행
2. README.md 업데이트 — README에 영향을 주는 변경일 때만
3. 커밋+푸시 — 위 단계 완료 후 사용자 요청으로

### 명시적으로 제외한 항목
- CHANGELOG 업데이트, 버전 범프 등은 마무리 시퀀스에 포함하지 않음
- 자동 마무리 감지는 채택하지 않음
