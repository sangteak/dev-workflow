# Phase 2: Discovery

## 1. 미정의 영역과 확정 결정

| 영역 | 결정 | 방식 |
|---|---|---|
| ① 진행 표시 | 결정(📋 확정 N/M)·질문(❓ 질문 N/M) **둘 다 유지, 별개 카운트** — "두 카운터는 별개로 센다" 1줄을 규칙에 추가 | 사용자 결정 |
| ② 기존 규칙 충돌 | 조사 에이전트가 약 30곳 전수 매핑 (아래 2절) — 대부분 기확정 결정의 기계적 반영, 갈림길 2건만 사용자 결정 | 조사 + 합의 + 사용자 결정 |
| ②-D1 §8 로컬 템플릿 | **전부 새 스켈레톤으로 개정** (약 10개 화면 — 시드 확인·Simplifier·국면0 카테고리·미답변 조사 Pass2·plan-stage 재협의·HANDOFF 삭제 확인·persona 저장 제안 등). "문서의 템플릿을 그대로 출력한다"는 §8 원칙 자체는 유지 | 사용자 결정 |
| ③ 질문 진행 상태 저장 | HANDOFF에 기존 결정 마커([x]/[>]/…)를 확장하지 않고 **가벼운 별도 줄** 신설: `❓ 질문 진행: [N]/[M] · 국면: [국면명]` + 미답변 질문 목록. 조사 판정: 기존 마커는 결정 전용 의미라 질문에 끼우면 의미 혼란 — 국면 1의 "마커 재사용" 합의를 이 방식으로 **수정** | 조사 + 합의 |
| ④ 규칙 문서 분리 | **한 파일 유지 (분리 부적합 판정)** — 규범 일체형(G4 판정 재적용), 분리 시 매 진입 2회 Read + 파일 간 불일치 재발 위험. 내부를 "결정 규칙/질문 규칙/공통 형식" 섹션으로 구분. 예상 크기 16KB → 24~33KB 감수 | 사용자 결정 |
