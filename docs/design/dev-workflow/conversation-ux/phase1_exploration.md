## 2. 확정 결정 (인터뷰 Step A, 5건)

1. **선답 처리 = 되돌린다**: 하나씩 모드에서 현재 질문의 답만 소비, 나머지는 순서대로 재질문. 선답을 모드 전환으로 자동 해석하지 않음 (명시적 "다 보여줘"는 유지)
2. **규범 위치 = 통합**: decision-flow.md를 결정+질문 공통 대화 규칙집(상호작용 SSOT)으로 확장. 질문(가볍게 무름)과 결정(기록·확정 절차)의 층위 구분은 문서 안에서 다룸
3. **형식 = 모양 고정 (사용자 지정 스켈레톤)**: 괘선(┌│└)·마크다운 표 금지. 질문(❓)도 같은 골격:
   ```
   [이모지] 결정 요청 : [제목]

   1. [선택지 — 고르면 어떻게 되는지 1줄]
   2. [선택지 — 고르면 어떻게 되는지 1줄]

   💡 추천: [번호]
   [사유 — 다음 줄]
   ```
4. **적용 범위 = 좁게**: dev-workflow 직접 진행 구간(BRAINSTORM/PLAN/COMPLETION + 세션 시작/컨텍스트 관리)만. Superpowers 위임 단계(DEVELOP/REVIEW)는 실제 불편 관측 후 확장 검토
5. **G5 관계 = 한 문장 명시**: "확인 횟수는 줄이되(G5 영역), 남은 확인은 하나씩 제시(이번 영역)"

## 3. 확정 결정 (Step C 확장 토론, 3건 + 정정 1건)

1. **선답 규칙 통일**: 결정(D)에도 동일 적용 — 원칙 "화면에 나온 항목에만 답할 수 있다". 하나씩 모드에서 다중 값 응답 시 첫 값만 소비+안내, 한꺼번에 모드(전체 노출)에서만 일괄 답변. **기존 decision-flow §4 일괄 응답 수용 조항을 이 원칙으로 개정** (v1.10.0 규범과의 충돌 첫 사례 판정)
2. **선택지 생략 판단 = 고정 표현 목록(화이트리스트)**: 사용자가 사실상 답을 말한 표현("그냥 기본값으로", "알아서 해줘", "당연히 A로")이 있을 때만 생략 허용. 모델 자유 판단 금지 (다중 모델 하드 제약)
3. **초심자 언어 = 기본값 (사용자 정정)**: 당초 "상시 최소 + 온디맨드 풀이" 2단 설계였으나 사용자가 정정 — 트리거 없이 기본값. 사용자 대상 질문·결정·결과 공유 문면의 기본: 짧은 배경 + 선택지별 결과 의미 + 용어 등장 시 풀이 + 💡 추천·사유 별도 줄 + 비유는 도움 될 때 (의무 아님). 발동 문구 목록 불필요

**세션 중 실시간 확정된 형식 규범 (사용자 직접 지정):**
- 추천 줄 분리: `💡 추천: [번호]` 다음 줄에 사유 (한 줄에 길게 잇지 않음)
- 요약·공유 출력: 구분 헤더 + `-` 항목 나열 + 구분 간 빈 줄
- 항목을 `/`로 한 줄에 이어 붙이지 않음 — 하나씩 `-` 줄로 나열

## 4. 페르소나 피드백 결과

- 확정 페르소나: 🛠️ Claude Code Expert / 🎯 Workflow Designer / 🔍 Ecosystem Analyst (국면 3 활성화)
- **[합의] Step C 라운드 1** [비판: 🎯 Workflow Designer — Contrarian]: 내부 모순 3건·누락 4건·약모델 리스크 3건·스코프 가드 3건 지적 → 사용자 결정 3건(위 3절)과 아래 합의로 해소:
  - 질문 "하나씩"은 brainstorming 기존 "한 번에 최대 2개" 규칙을 **대체** (교체를 변경 목록에 명시)
  - 질문 총 개수 M(❓ N/M)은 페르소나 전원 소집 후 확정
  - 답변된 탐색 질문의 답은 "잠정" — 국면 전환 시 확정 (phase 불변 구조와 정합)
  - 선택지 생략 오판 시 복구: 재논의 대기열 경유 없이 **즉시** 선택지 소급 제시
  - HANDOFF 저장: 기존 결정 흐름 마커 체계([x]/[>]/[ ]/[-]/[=]) 재사용, 모드는 저장하지 않음
  - 선답 처리는 기계적 규칙로 명문화: "하나씩 모드에서 쉼표/줄바꿈 구분 다중 값 → 첫 값만 현재 항목 반영 + 안내" (의도 해석 금지)
  - G5 관계 구체화: "G5는 확인의 묶음 단위를, 이번 규칙은 묶음 내부의 제시 방식을 다룬다" + 충돌 우선순위를 기존 규범 충돌 점검(국면 2)에 포함
  - decision-flow.md 파일 분리는 이번 feature에서 판정만, 실행 금지 (필요 시 별도 feature)
  - 요약 형식 규칙의 네거티브 스코프 문장 명시 (design-summary·merge-to-domain 기존 출력 형식은 별개)
