# Phase 3: Validation — Subagent Prompt Injection

> 검증 완료 시점의 불변 스냅샷

## TD 기술 검토 결과

### REQ-001: 기존 7개 템플릿 프레이밍 전환
- 판단: 텍스트 추가만, 기존 구조 불변
- 리스크: 낮음
- 가이드라인: `⚠️` 경고 + `===` 구분자 + "수정·요약·재구성하지 않고 파일 내용 그대로 포함" 문구 삽입

### REQ-002: Contrarian/Hacker 신규 템플릿
- 판단: 기존 4개 템플릿과 동일 구조로 확장
- 리스크: 낮음
- 가이드라인: Step C 맥락(시드 기반 확장 토론, 비판/교착)에 맞는 컨텍스트 섹션 포함

### REQ-003: 피드백 루프 일반 규칙 갱신
- 판단: 83행 일반 규칙에 "수정·요약 없이 그대로 포함" 명시
- 리스크: 낮음
- 가이드라인: 미래 에이전트에 대한 fallback 역할

### REQ-004: plan-stage, orchestrator 동일 적용
- 판단: brainstorming과 동일 패턴 기계적 적용
- 리스크: 낮음

## 재협의 발생 항목

없음. 모든 요구사항이 낮은 리스크로 통과.

## TD 기술 가이드라인

1. 프레이밍 전환 패턴은 `⚠️` + `===` 구분자 + 명시적 부정 제약으로 구성
2. 모든 에이전트 호출 템플릿에 동일 패턴 일관 적용
3. 일반 규칙(83행)도 갱신하여 fallback 보호
4. Contrarian/Hacker 신규 템플릿은 기존 패턴 + Step C 맥락 컨텍스트

## 페르소나 피드백 결과

- 전원 합의: 모든 항목 낮은 리스크, 기존 흐름 영향 없음
