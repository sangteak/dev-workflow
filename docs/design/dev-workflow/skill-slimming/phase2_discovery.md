# Phase 2: Discovery

## 1. 국면 2에서 발견된 미정의 영역

- 스킬별 must-preserve 불변식 체크리스트의 실제 항목 (결정 ②의 산출물) — 본 국면에서 구체화 완료

## 2. 확정된 불변식 체크리스트 구조

### 구조 규칙 (Contrarian 라운드 전면 수용)

- **3튜플 형식 강제**: 전 항목 `{파일, 앵커(정확 인용문 또는 정규식), 판정 방식(존재 grep / 바이트·해시 동일 / 개수 일치)}` — 앵커의 정확 인용문은 PLAN에서 베이스라인 추출로 확정
- **기계 체크 / 판단 리뷰 이원 목록**: 3튜플로 환원 불가능한 항목은 "판단 리뷰" 목록으로 분리 (신선한 눈 1회 교차 검토가 담당)
- **음성 불변식**: diff는 삭제·이동·파일 내 참조(§n) 삽입만 허용. 신규 규범 문장은 **별도 승인 목록**에 명시해야만 허용
- **파일 간 참조는 앵커 대체 불가** (파일 내 §참조는 허용) — F 공통 규칙

### 기계 체크 목록 (판정 방식 포함)

**A. brainstorming** (압축: 템플릿 공통부 통합+앵커, /compact 3중→1줄)
- A1. 피드백 루프 최대 3라운드 + `[라운드 N/3]` + **초과 시 "추가 토론이 필요할까요?" 승인 게이트** [존재 grep]
- A2. 라운드 로빈 비판 + `[비판: 페르소나]` 명시 [존재 grep]
- A3. 서브에이전트 디스패치 지점 **이름 열거 표** (피드백루프 비판/A-0 Ontologist/Step A Socratic/미답변 조사/Step B Seed-Architect/Step C Contrarian/Hacker/Simplifier/국면 4 Seed-Architect — 최대 9곳)에 지점별 ①역할 정의 로드 ②페르소나 도메인 ③출력 제약의 적용 여부 O/X 표를 베이스라인으로 확정 → 슬림화 후 O 항목 전수 잔존 [개수 일치 + grep]. ③ 결정 요청 금지 앵커는 **베이스라인에 있던 지점(피드백루프 비판 슬롯)만** 잔존 요구 — 신설은 승인 목록 경유
- A4. Step A: 해결책 제안 금지 + 최소 3회 Q&A + 전원 서브에이전트 [존재 grep]
- A5. Step B: 시드 사용자 확인 필수 + 명확도 체크리스트 [존재 grep]
- A6. Step C: 비판 슬롯만 서브에이전트 + Hacker 교착 투입 [존재 grep]
- A7. phase 파일 생성 시점·불변성·포함 내용 목록 [존재 grep]
- A8. /compact 1줄의 **확정 문면**: `💡 [phase파일명] 저장됨 — /compact 가능 (재논의 대기열은 먼저 처리 — decision-flow.md §6) · 새 세션: /clear → /dev-workflow:resume` — 3개 국면에 파일명만 바꿔 각 1회 [정확 문자열 3회]
- A9. 미답변 조사 2-pass + **A-0 변형의 분기 라우팅(조사 후 Step B 아닌 Step A로)** [존재 grep]
- A10. 각 📎 지점의 **Standalone 대응 문장** 잔존 (A-0→ESSENCE 대체 / Step A 4패턴+통합 정제 동일 / Step B 인라인 YAML / Simplifier 직접 분석) [존재 grep]
- A11. "TD는 발언하지 않는다" 3회 + "(탐색적 질문 한정 — …decision-flow…하나씩…)" 괄호 단서 2회 [개수 일치]

**B. decision-flow.md** (압축: §4 중복 2문장→§3 참조, §5 브리핑 재실행 조건→§8 참조)
- B1. N/M/K 기호 정의 + 표시 시점 3케이스 + **재산출 규칙 + 마지막 결정 후 헤더 미부착** [존재 grep]
- B2. 참조 치환 허용은 정확히 2문장(일괄→한꺼번에 전환, 위임→알아서 전환)만 — **그 외 §4 문장은 바이트 보존** (무관 주제 복귀 고지·지금/후에 분기·M 합산 포함) [해시 + 인용문 2개]
- B3. 결정 박스(D)·브리핑(F)·대기열 노출 박스 템플릿 [바이트 보존]
- B4. §8 참조 허용은 "브리핑 재실행 조건"만 — 재편입 M 카운터·'일부만' 미지정 잔여=기각·기각 phase 기록 3규정은 §5 문면 잔존 [존재 grep]
- B5. §6 불릿 4개 + "건수만 쓰지 않는다" + §9 표 + **§1(결정의 정의·순환 의존 유일 예외·클러스터 금지)** [존재 grep]
- B6. §4 '진행 중 목록 변경': 경량 이의 게이트 + "고지 없는 조용한 제거 금지" [존재 grep]
- B7. §8: 0/1/2+/8+ 분기 + 로컬 템플릿 우선(대상 3종 이름 포함) + 적용 예외 3항 [존재 grep]

**C. development-principles** (압축: 철학 4섹션→원칙 요약 블록)
- C1. 경로 해소 규칙 섹션(`## 경로 해소 규칙`~다음 `##` 직전, LF 정규화) [정규화 해시 동일]
- C2. 섹션별 grep 토큰 잔존: 단순함→"오버엔지니어링"·"최소", 근본 원인→"임시방편"·"시니어", 우아함→"우아한", 자율 수정→"CI" [존재 grep]
- C3. rules-injection·lessons 참조 잔존 [존재 grep]
- C4. lessons.md 기록 형식 템플릿 [바이트 보존] + "사용자에게 출력하지 않는다" [존재 grep]

**D. orchestrator** (압축: Evaluator 프롬프트/AC 박스 인라인 간소화)
- D1. Evaluator 프롬프트: 역할 정의 로드·수락 기준·**구현 결과물**·PASS/FAIL/PARTIAL 판정·**FAIL/PARTIAL 근거+수정 제안** 지시 + 동적 탐색 폴백 [존재 grep]
- D2. AC 박스 최소 필수 필드: PASS/PARTIAL/FAIL 3카운트 + 상세 목록 + `[Evaluator]` 태그 [존재 grep] — 간소화 허용 범위를 이 필드 집합으로 확정 (F1 바이트 보존 대상에서 제외 선언)
- D3. FAIL→재검증 제안 / 전부 PASS→진행 [존재 grep]
- D4. 스킵 가드 2개: Standalone→Evaluator 생략, 수락 기준 없음→스킵 [존재 grep]
- D5. 실행 순서: post-review-validate → Evaluator QA + "Success Criteria만 검증(중복 호출 회피)" [존재 grep]

**E. design-doc-index / design-summary** (압축: 상태 정규화 전처리→1줄)
- E1+E2. silent-fix 의미론(completed→complete, 허용 변형 completed만) — **두 파일에서 문자 동일** [정확 문자열 2회]
- E3. "정규화→필터링 순서" + domain.md 제외 잔존 [존재 grep]

**F. 공통**
- F1. 변경 허용 템플릿 **화이트리스트** (design-summary ━━→── 등 이름 지정 + 목표 스타일 문면) — 그 외 사용자 노출 템플릿은 빈 diff [해시]
- F2. 슬림화 전 번호 선택 게이트 grep **인벤토리**(게이트명·선택지 수·0번 유무)를 베이스라인으로 → 사후 동일 재현 [개수 일치]
- F3. 크로스레퍼런스 무결성: 전 파일 `참조|§` grep 인벤토리 → 각 참조의 대상 앵커 생존 확인 [기계 절차]
