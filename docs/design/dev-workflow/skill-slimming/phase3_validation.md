## 3. TD 기술 가이드라인

1. **작업 순서 강제**: ① 베이스라인 인벤토리 추출(3튜플 앵커·게이트·크로스레퍼런스) → ② 파일별 압축/분할 → ③ 기계 체크 전수 → ④ 판단 리뷰 1회. 순서 역전 금지
2. **분할 안전장치 3종**: (a) 분리 파일은 스킬 base dir 상대 경로(`Base directory for this skill` 제공 활용) (b) 각 참조 지점에 "Read하라" 명시 지시 + Read 실패 시 최소 인라인 폴백 (c) 분리 파일을 F3 크로스레퍼런스 인벤토리에 포함
3. **분리 파일 명명**: `skills/[스킬]/references/[주제].md` (Superpowers references/ 관행 준용)
4. **음성 불변식**: diff는 삭제·verbatim 이동·참조 삽입만. 신규 규범 문장은 승인 목록 경유 (현재 승인 목록: 없음 — 앵커 신설도 미승인)
5. 다중 모델 제약 유지: 분리 파일 Read 지시는 약한 모델도 따를 수 있게 디스패치 지점마다 명시 (참조 1회 정의로 갈음 금지)
