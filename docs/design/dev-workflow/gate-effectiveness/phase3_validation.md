## 3. TD 기술 가이드라인

1. **구 독트린 전수 스윕** (conversation-ux 교훈 계승): 의미가 역전되는 규범("생략할 수 없다"·"학습 게이트"·"커밋+푸시" 계열)은 grep 인벤토리로 잔존 문장을 전수 스윕 — 무변경 컨텍스트 줄의 옛 규범이 태스크 리뷰 사각임이 실증된 바 있음
2. **차단성 우려 판별문 고정** (판단형 금지): "이 우려가 반영되지 않으면 dry-run 승인을 진행할 수 없는가?" — Yes=차단(라운드 증가), No=권고(dry-run 회계 1줄)
3. **커밋/푸시 분리와 bootstrap 동기화는 같은 태스크** — 폴백 누락 선례 재발 방지
4. **3단 결정 트리 문면을 development-principles 신설 절에 그대로 수록** — 이번 심사와 미래 심사(게이트 신설 체크리스트)가 같은 기준 공유
5. 검증 체계: 게이트 전후 대조 목록 + 확정 문면 grep + 드라이런 1회 (가상 마무리 시나리오 — 개정 문면만 읽는 신선 에이전트) — 신규 인프라 금지
6. 도메인 문서(document-management.md 등)의 게이트 서술 갱신은 이번 범위 밖 — 본 기능의 도메인 머지 시점에 수행
