# Phase 2: Discovery

## 1. 미정의 영역과 진행 방식

국면 1 이월 3건(심사 실행 · 형식 게이트 3건 처분 · 파일 매핑)을 조사 에이전트 2개(3단 트리 기계 심사 / 형식 게이트 심층+매핑)로 병렬 조사 후, 갈림길 6건을 사용자 결정으로 확정했다.

## 2. 게이트 전체 목록 보정 (phase1 §6에 대한 정정 — phase1은 불변이므로 여기 기록)

- 형식적 게이트 관측 ②(Architect 자동 에스컬레이션)·③(국면 4 2차 Seed-Architect)은 phase1 표에 행으로 미등재였음 — 본 국면에서 게이트 행으로 등재하고 처분 확정 (아래 4절). ②는 merge-to-domain 라운드 규정+plan-stage Step 4 강제 규정, ③은 게이트가 아닌 자동 절차의 형식적 반복
- 계수 명확화: 표는 "게이트 33건 + 참고 1행(Completion Step1, 확인 없음) = 34행"으로 읽는다
- 실행 모드 선택(#27)의 가역성 공란은 "가역"으로 보완 (처리 방식 차이일 뿐 산출물 손실 없음)

## 3. 기계 심사 결과 (3단 결정 트리 적용 — 사용자 수용 확정)

**집계: 유지 22 · 자동화 5(전부 안전망 확보) · 제거 0 · 분리 1 · 조건부 1 · 별도 결정 3**

### 자동화 5건 (확인 생략 + 안전망 동반 — 구체 문면은 PLAN에서 확정)

| 게이트 | 안전망 |
|---|---|
| document-consolidation 중간 산출물 삭제 확인 | 삭제 내역 사후 고지 1줄 + git 이력 (거부 경로: 고지 후 이의 시 복원) |
| brainstorming 국면 4 후 "PLAN으로 진행할까요?" | 자동 진행 고지 1줄 (설계 문서는 이미 저장 완료 — 거부해도 잃는 것 없음) |
| context-handling HANDOFF 복구 재확인 | 목록 선택 자체가 의사 표명 — 복구 시작 고지 1줄, HANDOFF 파일 보존으로 되돌림 가능 |
| persona-resolution 페르소나 저장 제안 | 자동화 형태는 PLAN에서 설계 (거부 결과가 "다음에 재질문"뿐인 확인) |
| merge-to-domain 자동모드 키워드 confirm | 오탐 시 "아니라고" 답할 길 유지 — 진입 고지 1줄, 의미 충돌은 별도 게이트가 재질문 |

### 유지 22건 요지

내용이 매번 다르거나(시드 확인·재협의 4지·의미 충돌 등) 비가역(국면 전환의 불변 파일 생성·dry-run 승인·no-git 삭제 등)이어서 확인이 실제 보호를 제공하는 게이트 전부. 제거 판정 0건 — "게이트 수 줄이기가 목적이 아니다"(phase1 비목표)와 정합.

## 4. 사용자 확정 결정 (6건)

- [x] 기계 심사 결과 수용 — 확정: 유지 22 + 자동화 5, 계 27건 처분 확정
- [x] 커밋/푸시 분리 — 확정: **커밋 확인 + 푸시 확인 2단 분리** (커밋 자동화안은 사용자 전역 CLAUDE.md "커밋 전 반드시 확인"과 충돌하여 기각)
- [x] 잔존 HANDOFF 삭제 — 확정: **git 프로젝트만 자동 삭제+고지 1줄**, no-git 프로젝트는 확인 유지 (환경 조건부 — 기계 심사의 유일한 "애매" 해소)
- [x] digest 학습 게이트 — 확정: **생략 + 사후 고지 1줄** ("digest 추출: 정책 N·결정 M건"). 근거: 화면이 원문 근거 없이 요약만 제시해 검증 불가(잡을 재료를 안 줌), 안전망은 이미 이중(생략 불가 dry-run + 문장 대조 검증). "생략 불가" 문면 2곳(SKILL.md:283·347) 개정 필요
- [x] Architect 라운드 — 확정: **포괄 정비** — ① 등급표 빈칸 보수("사용자 결정 3건+ · 의미 충돌 0 → 2라운드" 행 추가) ② 라운드 증가는 차단성 우려(반영 없이 진행 불가)에만 발동, 가벼운 권고는 dry-run 회계에 1줄 기록 ③ plan-stage는 페르소나 전원 일치 시 합의 기록 후 라운드 종료
- [x] 국면 4 시드 재생성 — 확정: **직접 갱신 전환** — 에이전트 재투입 없이 Step B 확정 시드에 국면 2~4 변경분만 반영해 seed.yaml 작성 (재투입의 설계 근거 부재 확인 — thinking-enhancement.md 매칭 표에 미등재, 실소비처는 document-consolidation 1곳)
